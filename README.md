## Задание №2
*Доказуемо простые числа, как правило, используются в качестве
модулей криптосистем, основанных на проблеме дискретного
логарифмирования, таких как шифр Шамира, криптосистема Эль-Гамаля и
связанные с ней стандарты цифровой подписи ГОСТ Р 34.11-94, ГОСТ Р 34.11-
2001, DSA и EСDSA. Рассмотрим тесты на простоту Миллера, Поклингтона,
ГОСТ.*
*В данном стандарте для цифровой подписи представлен алгоритм генерации простых чисел, основанный на теореме Диемитко.
Теорема Диемитко. Пусть 𝑛 = 𝑞𝑅 + 1 > 1, где q – простое, R – четное и 𝑅 < 4(𝑞 + 1). Тогда n – простое число, если найдется такое целое число a, что выполнены следующие условия:
1) 𝑎^(𝑛−1) ≡ 1 (𝑚𝑜𝑑 𝑛);
2) 𝑎^((𝑛−1) / q !=  1 (𝑚𝑜𝑑 𝑛).
Входные данные: t – требуемая разрядность числа (совпадает с bits), q – простое число, разрядность которого равна ⌈𝑡/2⌉ (округление до целых в большую сторону).
Процедура генерации числа в ГОСТ Р.34.10-94:
1. Сгенерировать случайную величину 𝜉, 0 < 𝜉 < 1;
2. Вычислить число N по следующей формуле:*

Если N – нечетное, то 𝑁 ≔ 𝑁 + 1;
3. Задать число 𝑢 = 0;
4. Вычислить число 𝑝 = (𝑁 + 𝑢)𝑞 + 1 – кандидат в простые числа;
5. Если 𝑝 > 2^𝑡, то вернуться на шаг 1;
6. Если выполняются условия теоремы Диемитко, т.е. выполняются
